# 底稿小幫手 — 維護手冊

**適用對象：** 接手維護本插件的同事
**技術需求：** 不需要程式背景，依照本手冊操作即可

---

## 一、這個插件是什麼組成的？

打開資料夾你會看到以下檔案：

```
網頁插件/
├── manifest.json        ← 插件的基本設定（名稱、版本、權限）
├── popup.html           ← 插件的畫面結構
├── popup.css            ← 插件的視覺樣式（顏色、字體、排版）
├── popup.js             ← 插件的所有功能邏輯
├── icons/               ← 插件圖示
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
└── 底稿小幫手_v2.0.zip  ← 打包好的上架檔案
```

**你最常需要動到的只有兩個地方：**
- `popup.js` — 修改文字、功能、版本記錄
- `popup.css` — 修改顏色、大小

---

## 二、如何在電腦上測試插件

每次改完程式碼，要這樣重新載入才會看到效果：

1. 開啟 Chrome，在網址列輸入：`chrome://extensions`
2. 找到「底稿小幫手」
3. 點擊右下角的「重新整理」按鈕（循環箭頭圖示）
4. 點開插件圖示，確認改動是否正確

> ⚠️ 每次改完都要重新整理才會生效，不會自動更新。

## 三、常見維護工作

### 3.1 修改版本號

**需要動的地方有三個：**

**① `manifest.json` 第 3 行**
```json
"version": "2.0",   ← 改這裡
```

**② `popup.html` 第 19 行**
```html
<button ...>v2.0</button>   ← 改這裡（顯示在畫面右上角）
```

**③ `popup.js` — RELEASE_LOG（版本記錄內容）**
找到 `const RELEASE_LOG = [` 這一行，在最上方新增一個版本區塊（格式見下方說明）。

---

### 3.2 新增版本記錄

在 `popup.js` 找到 `const RELEASE_LOG = [` 這一行，在第一個 `{` 之前插入新版本：

```js
const RELEASE_LOG = [
  {
    version: '2.2',           // ← 版本號
    date: '2026-05-01',       // ← 日期
    badge: '功能更新',         // ← 標籤文字（可改或刪掉這行）
    groups: [
      {
        label: '新功能',       // ← 分類標題
        items: [
          '新增了某某功能',     // ← 每一條改動
          '修正了某某問題',
        ]
      },
    ]
  },
  {
    version: '2.0',           // ← 舊版本保持不動，往下放
    ...
```

> 每個版本之間用逗號 `,` 隔開，注意不要漏掉。

---

### 3.3 修改顏色

所有顏色都在 `popup.css` 裡面。本插件有兩個主要顏色：

| 用途 | 色碼 | 說明 |
|------|------|------|
| 主色（黑色 header） | `#1a1a1a` / `#000000` | 頂部標題列背景 |
| 副視覺色 | `#177077` | 按鈕、高亮、標籤等強調色 |
| 按鈕藍色 | `#2c7cb0` | 儲存按鈕、標籤顏色 |

如果要換副視覺色，用文字編輯器的「全部取代」功能，把所有 `#177077` 換成新色碼即可。

---

### 3.4 修改搜尋框的提示文字

在 `popup.html` 找到：
```html
<input type="text" id="search-input" placeholder="搜尋文字..." />
```
把 `搜尋文字...` 改成你要的文字。

---

### 3.5 修改插件名稱或描述

在 `manifest.json`：
```json
"name": "底稿小幫手",
"description": "工作底稿常用文字管理工具，一鍵複製你需要的文字！",
```

> ⚠️ 名稱改了之後要重新上架商店審核。

## 四、如何打包上架

### 4.1 打包壓縮檔

打包只需要包含以下檔案（不要把說明文件、截圖等一起打包進去）：

- `manifest.json`
- `popup.html`
- `popup.css`
- `popup.js`
- `icons/` 資料夾（含三個 png）

打包方式：
1. 選取以上檔案
2. 右鍵 → 壓縮
3. 重新命名為 `底稿小幫手_vX.X.zip`

或請原開發者幫你用指令打包。

### 4.2 上架到 Chrome Web Store

1. 前往 [Chrome Web Store 開發者後台](https://chrome.google.com/webstore/devconsole)
2. 登入開發者帳號
3. 找到「底稿小幫手」→ 點擊「管理」
4. 點擊「上傳新版本」→ 選擇打包好的 zip 檔
5. 填寫版本說明 → 送出審核

> 審核通常需要 1–3 個工作天。

## 五、資料說明

### 使用者的資料存在哪？

存在 Chrome 的本機儲存空間，路徑為：
- **Mac**：`~/Library/Application Support/Google/Chrome/Default/Local Extension Settings/<插件ID>/`
- **Windows**：`%LOCALAPPDATA%\Google\Chrome\User Data\Default\Local Extension Settings\<插件ID>\`

**更新版本不會清除使用者資料。**
只有使用者自己解除安裝插件才會清除。

### data_template.json 是什麼？

這是提供給使用者的 JSON 範本，讓他們可以預先填好自己的文字資料再匯入插件，不需要從零開始一筆一筆新增。

## 六、程式碼版本管理（Git）

本專案使用 Git 做版本控制，所有改動都有記錄。

### 查看歷史改動
```
git log --oneline
```
會列出所有曾經做過的改動紀錄。

### 還原到某個舊版本
如果改壞了想回到之前的版本：
```
git checkout <版本代號>  ← 版本代號是 log 列出的那串英數字
```

### 儲存目前的改動
```
git add popup.js popup.html popup.css
git commit -m "說明這次改了什麼"
```

> 不確定怎麼操作 Git，可以請原開發者協助或使用 Claude Code 工具輔助。

---

## 七、緊急狀況處理

### 改壞了怎麼辦？

**方法一：用 Git 還原**
```
git restore popup.js
```
這會把檔案還原成上次 commit 的狀態。

**方法二：從舊版 zip 取出**
從之前打包好的 `底稿小幫手_vX.X.zip` 解壓縮，取出原本的檔案覆蓋。

**方法三：聯絡原開發者**
提供 Git commit 記錄，原開發者可以快速找到問題所在。
